What is a SOC?

A Security Operations Center (SOC) is a team responsible for continuously monitoring, detecting, investigating, and responding to cyber security threats.
SOC Level 1 (Junior Security Analyst)
Responsibilities
Monitor security alerts
Investigate alerts
Escalate complex incidents
Document findings
Work in shifts (often 24/7)
Continuously learn new attack techniques
Typical Daily Tasks
Review security alerts
Investigate suspicious activity
Cooperate with other security teams
Attend SOC meetings/workshops
Learn new threats and defenses
SOC Team Roles
Junior Security Analyst (SOC L1)
First line of defense
Reviews alerts
Performs initial investigation
Escalates when necessary
Senior Security Analyst (SOC L2/L3)
Handles advanced investigations
Mentors junior analysts
Performs deep analysis
SOC Engineer
Configures SIEM tools
Creates detection rules
Tunes alerts
Maintains security infrastructure
SOC Manager
Manages the SOC team
Reports to executives
Oversees operations
Incident Responder
Responds to major incidents
Contains and eradicates attacks
Leads incident response
Common Incidents
Phishing
Malware
Data Stealers
Ransomware
DDoS attacks
Supply Chain attacks
SaaS compromises
Important Concepts
SOC = Security Operations Center
Alert = Notification generated by a security tool
Ticket = Investigation assigned to an analyst
Escalation = Passing a complex case to a higher-level analyst
Key Takeaways
SOC analysts are the first line of cyber defense.
SOC operates 24/7.
Continuous learning is essential.
Teamwork is critical.
Not every alert is an attack—investigation is required.
